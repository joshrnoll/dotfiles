# Define Outcome and Plan

## Use this when

- the problem is understood
- likely solutions are narrowed enough to define a direction
- the user wants a clear desired outcome and high-level approach

## What I will do

- ask clarifying questions as needed
- clearly articulate the desired outcome
- define success criteria
- establish scope and non-goals
- describe the high-level approach

## What I will not do

- drop into low-level implementation steps
- produce a detailed execution checklist
- begin implementation

## Completion criteria

This skill is complete only when:
1. the desired outcome is explicitly stated and approved
2. the topic name has been proposed and approved
3. `workflow/<topic>/PLANNING.md` has been written
4. the desired outcome appears clearly at the top of the file

## Artifact output

Write `workflow/<topic>/PLANNING.md` with these sections:
- Desired outcome
- Success criteria
- Scope
- Non-goals
- High-level approach
- Risks / assumptions
